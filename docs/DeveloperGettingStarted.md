# :wave: Welcome to the Dev Hub! @Developers 

Welcome aboard! This guide will help you get up and running with our codebase, tooling, and workflows. Follow the steps below to start contributing.

━━━━━━━━━━━━━━━━━━━━━━

## :open_file_folder: Step 1: Access the Repository

Make sure you have a GitHub account linked with our organization.

**Repository:** [https://github.com/Avstn1/shearworkWEB)

Clone locally:

```bash
git clone https://github.com/your-org/your-repo.git
cd shearwork-web
```

━━━━━━━━━━━━━━━━━━━━━━

## :gear: Step 2: Install Dependencies

We use **Node.js** and **npm** (or yarn). Check your versions:

```bash
node -v   # Recommended: v20.x
npm -v    # Recommended: v10.x
```

Install dependencies:

```bash
npm install
# or
yarn install
```

━━━━━━━━━━━━━━━━━━━━━━

## :tools: Step 3: Start the Development Server

Run the development server:

```bash
npm run dev
# or
yarn dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser to see the app.

━━━━━━━━━━━━━━━━━━━━━━

## :book: Step 4: Useful Documentation

* Project Wiki: [https://github.com/Avstn1/shearworkWEB](https://github.com/Avstn1/shearworkWEB)
* Coding Guidelines: /docs/CODING_GUIDELINES.md

━━━━━━━━━━━━━━━━━━━━━━

## :twisted_rightwards_arrows: Step 5: Branching & Workflow

* Use **feature branches**: `feature/<your-feature>`
* Create a **pull request** for all changes
* Follow commit message convention: `type(scope): short description`
  Example: `feat(auth): add login endpoint`

━━━━━━━━━━━━━━━━━━━━━━

## :bulb: Tips & Best Practices

* Check open issues before starting new features
* Run `npm test` or `yarn test` to verify your changes
* Keep commits small and descriptive
* Ask questions in `#dev-help` if you get stuck

━━━━━━━━━━━━━━━━━━━━━━

## :tada: You’re All Set!

Once you’ve completed the steps above, you’re ready to start contributing. Happy coding! :rocket:
